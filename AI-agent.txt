Phase 0: 项目初始化

Vite 项目、全部依赖、Tailwind、Element Plus、目录结构、代理配置(/api -> http://localhost:8000)、TypeScript 均已就绪。跳过此阶段。

Phase 1: 

 utils/request.ts(JWT 拦截器)、store/auth.ts、api/auth.ts、路由守卫(router/index.ts)

LoginView.vue -- 登录表单(用户名+密码),需验证 UI 完整性

RegisterView.vue -- 注册表单(用户名+邮箱+密码+确认密码),需验证表单校验

api/auth.ts -- 确认 login/register/getMe 接口定义与后端对齐

store/auth.ts -- 确认 token 持久化(localStorage)和 user 状态管理

路由守卫 -- 确认未登录跳转 /login,已登录访问 /login 跳 /chat

验收标准: 能完成注册->登录->进入主页面的完整闭环,token 正确存储和携带。


Phase 2: 整体布局与导航 -- 需检查完善度

AppHeader.vue -- Logo + 导航 Tab(聊天/画布/知识库/工具) + 用户下拉菜单

MainLayout.vue -- Header 固定顶部,下方 <router-view> 内容区


用户下拉菜单 -- 个人信息、退出登录功能

验收标准: 顶部导航可用,Tab 切换对应路由,用户菜单可退出登录。 （此阶段已全部完成，实现了可以自由切换的顶部导航，并且用户可以退出登录）

Phase 3: 聊天界面  （已全部完成，可以实现自由对话并保存到数据库内）

目标: 完整对话体验,支持 SSE 流式输出。

ChatView.vue -- 页面容器(左侧对话列表 + 右侧聊天区)

components/chat/ConversationList.vue -- 对话历史列表 + 新建对话按钮

components/chat/MessageList.vue -- 消息滚动列表,自动滚到底部

components/chat/MessageItem.vue -- 单条消息(区分用户/AI/系统,Markdown 渲染)

components/chat/ChatInput.vue -- 输入框 + 发送按钮 + 附件按钮

components/common/MarkdownRenderer.vue -- AI 回复 Markdown 渲染 + 代码高亮(使用 marked + highlight.js)

utils/sse.ts -- SSE 连接管理(逐字推送,打字效果)

api/chat.ts -- sendMessage / getConversations / getMessages 接口实现

store/chat.ts -- 当前对话、消息列表、流式状态管理

关键数据流: 用户输入 -> store.chat.sendMessage -> POST /api/chat -> SSE 逐条消息 -> store.chat.messages 实时更新 -> MessageList 渲染


Phase 4: 画布编辑器 -- 核心（！！！！！！！！！！！！！）

目标: 可视化工作流搭建,节点拖拽+连线+配置。

待完成:

CanvasView.vue -- 画布页面容器(三栏布局: 左节点面板 | 中画布 | 右配置面板)

NodePalette.vue -- 左侧节点拖拽源面板(起始/LLM/RAG/工具/条件/输出)

WorkflowCanvas.vue -- Vue Flow 画布主组件(注册节点类型、处理拖拽放置、连线)

6 个节点组件(StartNode/LLMNode/RAGNode/ToolNode/ConditionNode/EndNode) -- 各自的 UI 和 Handle 定义

NodeConfigPanel.vue -- 右侧动态表单,根据选中节点类型切换配置项

store/canvas.ts -- 节点列表、连线、选中节点状态

api/workflow.ts -- saveWorkflow / loadWorkflow / listWorkflows 接口

types/workflow.ts -- 节点类型、连线类型、工作流类型定义

Vue Flow 集成要点:

使用 @vue-flow/core 的 VueFlow 组件

通过 useVueFlow() composable 管理节点/边状态

自定义节点通过 nodeTypes 选项注册

拖拽放置通过 onDrop 事件 + addNodes 实现
